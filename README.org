#+CATEGORY: org-glance
#+STARTUP: overview

A personal knowledge graph for [[https://orgmode.org/][org-mode]].  Headlines become nodes in a
durable plain-text store; typed references between them become edges;
fast and flexible views show it all without opening a single file.

* Install

The published package archive is the simplest route.  Add the archive, then let
=use-package= install and set it up:

#+begin_src emacs-lisp
(add-to-list 'package-archives
             '("rails-to-cosmos" . "https://rails-to-cosmos.github.io/elpa/"))

(use-package org-glance
  :bind (("C-x j" . org-glance-transient))
  :config (org-glance-init)
  :ensure t)
#+end_src

=C-x j= opens =org-glance-transient=, the transient dispatcher (see
[[*Highlights][Highlights]]); =org-glance-init= bootstraps the store.

The =rails-to-cosmos= archive provides =org-glance= and its
[[https://github.com/rails-to-cosmos/table-view][table-view]] and
[[https://github.com/rails-to-cosmos/agnostic-llm][agnostic-llm]] dependencies; the
remaining deps (=dash=, =f=, =s=, =aes=, =transient=) come from
MELPA, so keep a MELPA entry in =package-archives= too.  Everything else (=org=
and its submodules, =cl-lib=, =thunk=, =subr-x=) ships with Emacs 29.1+.

The *LLM session* action (=l=) opens an [[https://github.com/rails-to-cosmos/agnostic-llm][agnostic-llm]] session in the
selected headline's data directory -- or in its project directory,
when the headline carries an =:ORG_GLANCE_PROJECT_DIR:= drawer property
(set it from a materialized buffer with =C-c d=,
`org-glance-material:set-project-dir').  [[https://github.com/rails-to-cosmos/agnostic-llm][agnostic-llm]] (also on the
=rails-to-cosmos= archive) is a required dependency; it pulls in =vterm=,
loaded lazily the first time you run the action, so org-glance never
loads vterm at startup.

* How it works

If you know org-mode, you know almost everything already:

1. *Headlines are the unit.*  Every captured headline lives as its own small
   org file in a hidden store (=.org-glance/= under =org-glance-directory=),
   keyed by an =:ORG_GLANCE_ID:= property org-glance adds for you.  No
   agenda-files list, no ever-growing monolith files.
2. *Tags are collections.*  An ordinary org tag (=:book:=, =:coffee:=) groups
   headlines; a tag exists the moment you capture something with it — no
   declaration step.  Optionally give a tag its own todo cycle and capture
   template (=C= in the transient).
3. *Views are projections.*  The table and the overview are generated,
   read-only summaries built from an index — fast, and always rebuildable.
   You never edit a view: you open the headline behind it, edit ordinary org,
   save; views refresh themselves.
4. *Links are edges.*  A link from one headline to another is indexed as a
   typed relation, so the graph can answer "what references this?".

* Usage, step by step

Everything is reachable from =org-glance-transient= (=C-x j=).

** 1. Capture a headline

=C-x j += → pick (or type) a tag → an org-capture-style buffer opens →
write an ordinary headline, =C-c C-c=:

#+begin_src org
,* TODO Kebena Decaf :coffee:
SCHEDULED: <2026-08-01 Sat>
Light roast, very floral.
#+end_src

** 2. Browse a tag

=C-x j o= → pick a tag → its *table*: one row per headline, sortable (=^=),
filterable (=/=), with colour-coded todo states.  =T= toggles to the org-text
*overview* — a plain org buffer, so folding, =C-x j a= (agenda) and link
clicking all behave exactly as in org.  Each overview heading is
self-sufficient:

#+begin_src org
,* TODO [#A] Kebena Decaf :coffee:
DEADLINE: <2026-08-10 Mon> SCHEDULED: <2026-08-01 Sat>
- roasted by [[org-glance-material:0e904834][Manhattan Coffee Roasters]]
- [[https://example.com][Homepage]]
#+end_src

Act on the row/heading at point without opening anything: =C-c C-t= todo
(with marks: bulk, org's own fast-selection prompt) · =C-c C-s= / =C-c C-d=
schedule / deadline · =:= add a tag · =-= drop the view's tag · =D= delete ·
=j= open a link inside it · =e= copy a =KEY: value= from its body.

In the table, columns are yours: =C-u += adds one for any drawer property or
relation kind the visible headlines carry (=Roasted by= → target titles),
=C-u -= removes any column but Title; layout persists per tag.  =?= shows the
key legend.

** 3. Edit a headline

=RET= on a row/heading (or =C-x j m= anywhere) *materializes* it: its file
opens in a normal org buffer.  Everything org can do works here — this IS
org-mode.  =C-x C-s= saves and re-indexes; open views refresh on their next
display.  (The =:ORG_GLANCE_ID:= drawer is concealed and protected: hand
edits to it revert on save with a warning.)

** 4. Connect headlines

In a materialized buffer, =@= at the start of a word prompts for another
headline and inserts a link to it; =C-u @= also asks a reference *kind*:

#+begin_src org
roasted by [[org-glance-material:0e904834?kind=roasted-by][Manhattan Coffee Roasters]]
#+end_src

Clicking the link opens the roaster.  The graph indexes it as a typed edge:
=C-c @= tables everything this headline references, =C-u C-c @= everything
that references it, and the kind can become a table column (step 2).  A link
to a whole view also exists: =[[org-glance-overview:coffee?done=nil][Active coffees]]=.

** 5. Keep secrets

Wrap a region in a crypt block with =C-c #= (no region: the whole body):

#+begin_src org
,#+begin_crypt
bank: 1234-5678
,#+end_crypt
#+end_src

On disk it is ciphertext; in the materialized buffer it is plaintext
(re-sealed on every save; auto-save and backups are disabled).  Text OUTSIDE
the blocks stays public and searchable.  =C-u C-c #= unwraps a block;
=C-c #= again forgets the cached password (TTL
=org-glance-material-password-ttl=).

** 6. Repeating headlines keep their history

A headline with a repeater (=SCHEDULED: <… +1d>=) stays ONE headline.  With
=org-glance-repeat-history-depth= set (N newest, =t= = all, per-headline
=:ORG_GLANCE_REPEAT_HISTORY_DEPTH:= wins), each completion snapshots the done
state before org resets it; the table's =Rep= column marks repeaters, and
=l= (=C-c l= in the buffer) opens any past occurrence read-only.

** Customization

| Variable                                | Default              |                                                        |
|-----------------------------------------+----------------------+--------------------------------------------------------|
| =org-glance-directory=                  | =org-directory=      | root of the managed content (store lives in =.org-glance/=) |
| =org-glance-overview-default-view=      | =org-glance-table=   | what =o= opens; =org-glance-overview= for org-text     |
| =org-glance-filter-spec=                | =(:done nil)=        | ambient filter every view starts from (active-only)    |
| =org-glance-view-fill-frame=            | =t=                  | views take the whole frame; =nil= keeps your windows   |
| =org-glance-repeat-history-depth=       | =0=                  | occurrence snapshots kept: N / =t= = all / =0= = off   |
| =org-glance-material-password-ttl=      | =300=                | seconds a crypt password stays cached; =0= = forever   |
| =org-glance-material-hidden-properties= | id + hash            | drawer keys concealed AND edit-protected in material buffers |
| =org-glance-table-state-colors=         | tokyo-night-ish      | per-state badge colours (=…-default-state-color= for the rest) |
| =org-glance-conflict-resolution=        | =ask=                | git-conflict healing: =ask= / =union= / =nil= (error)  |
| =org-glance-tag-config-file=            | =nil=                | override path of =config/tags.org=                     |
| =org-glance-graph-segment-max-bytes=    | 256k                 | WAL segment size before sealing                        |
| =org-glance-graph-compact-segment-count= | =4=                 | sealed segments that trigger auto-compaction           |

Example — org-text views, everything visible, daily history for a year:

#+begin_src emacs-lisp
(setq org-glance-overview-default-view 'org-glance-overview
      org-glance-filter-spec nil               ; show DONE too
      org-glance-repeat-history-depth 365)
#+end_src

** The store, briefly

A segmented, append-only JSONL log with atomic commits, compaction, crash
recovery and git-sync conflict healing.  The per-headline org files are
canonical; every index is derived and rebuildable (=M-x org-glance-reindex=).
Deletion (=D=, =M-x org-glance-delete=) writes a tombstone after a
confirmation that names any headlines whose links would dangle.

* Install from source (development)

For hacking on org-glance, clone the repo and load it directly.  It is a
multi-directory package: =org-glance.el= is at the root, modules under =src/data=
and =src/view=; it does NOT add those to =load-path= itself, so all three must be
on it.  [[https://github.com/rails-to-cosmos/table-view][table-view]] and
[[https://github.com/rails-to-cosmos/agnostic-llm][agnostic-llm]] are external
dependencies from the =rails-to-cosmos= ELPA archive (not on MELPA); the MELPA
deps above are still needed.  (=Eask= is the dev/test harness, not required to use the package.)

#+begin_src emacs-lisp
(dolist (dir '("" "src/data" "src/view"))
  (add-to-list 'load-path (expand-file-name dir "~/path/to/org-glance")))
(require 'org-glance)
#+end_src

With =straight.el= (also needs recipes for
[[https://github.com/rails-to-cosmos/table-view][table-view]] and
[[https://github.com/rails-to-cosmos/agnostic-llm][agnostic-llm]]):

#+begin_src emacs-lisp
(use-package org-glance
  :straight (org-glance :type git :host github :repo "rails-to-cosmos/org-glance"
                        :files ("*.el" "src/data/*.el" "src/view/*.el")))
(use-package table-view
  :straight (table-view :type git :host github :repo "rails-to-cosmos/table-view"))
(use-package agnostic-llm
  :straight (agnostic-llm :type git :host github :repo "rails-to-cosmos/agnostic-llm"))
#+end_src
