Emacs front-end for [[https://github.com/rails-to-cosmos/glance][Glance]]. Work in progress.

* Install

The published package archive is the simplest route.  Add the archive, then let
=use-package= install and set it up:

#+begin_src emacs-lisp
(add-to-list 'package-archives
             '("rails-to-cosmos" . "https://rails-to-cosmos.github.io/elpa/"))

(use-package org-glance
  :bind (("C-x j" . org-glance-transient))
  :config (org-glance-init)
  :ensure t)
#+end_src

=C-x j= opens =org-glance-transient=, the transient dispatcher (see
[[*Highlights][Highlights]]); =org-glance-init= bootstraps the store.

The =rails-to-cosmos= archive provides =org-glance= and its
[[https://github.com/rails-to-cosmos/table-view][table-view]] dependency; the remaining deps (=dash=, =f=, =s=, =aes=, =transient=) come from
MELPA, so keep a MELPA entry in =package-archives= too.  Everything else (=org=
and its submodules, =cl-lib=, =thunk=, =subr-x=) ships with Emacs 29.1+.

The *LLM session* action (=l=) opens an
[[https://github.com/rails-to-cosmos/agnostic-llm][agnostic-llm]] claude session in the
selected headline's data directory -- or in its project directory, when the
headline carries an =:ORG_GLANCE_PROJECT_DIR:= drawer property (set it from a
materialized buffer with =C-c d=, `org-glance-material:set-project-dir').
[[https://github.com/rails-to-cosmos/agnostic-llm][agnostic-llm]] (also on the
=rails-to-cosmos= archive) is a required dependency;
it pulls in =vterm=, loaded lazily the first time you run the action, so
org-glance never loads vterm at startup.

* Highlights

Reach everything through the =org-glance-transient= transient (=C-x j= above).

- *Overview* (=o=) — the org-text overview of a chosen tag: each heading is
  self-sufficient (priority + planning for =org-agenda=, clickable body links,
  relations with kinds and live titles — no materialization needed); toggle it (=T=) to a
  sortable headline *table* with custom drawer-property columns and bulk state
  change.  =C-u += adds and =C-u -= removes a table column — a drawer property the
  view's headlines carry for add (completion, backed by a persisted property
  index), any column but the mandatory Title for remove; both persist per tag.
  =:= adds a tag to the row (offering only tags it lacks), a bare =-= drops the
  view's tag off it.  The table header always shows the sort and a coloured
  =#+TODO:= line of the graph's states; =?= toggles the action-key legend
  (RET:Materialize …).
- *All tags* (=t=) — a table of the tags THEMSELVES: per-tag headline count, todo
  state breakdown, configured cycle, and event-tracked created / last-modified
  timestamps.  State and Cycle cells are colour-coded.  In the buffer: =RET=
  opens that tag's headline table, =o= its overview, =+= adds a tag (captures a
  first headline), =-= removes a tag (a guarded, non-destructive retag).
- *Agenda* (=a=), *Capture* (=+=), *Extract property* (=e=), *Open link* (=j=),
  *Materialize* (=m=), *LLM session* (=l=), *Configure tags* (=C=).

Other features:

- *References* — in a materialized buffer, =@= at a word boundary links another
  headline (completion, required match; mid-word =@= just types); =C-u @= adds a
  reference *kind* (a book's =author= vs =editor=).  The inserted
  =[[org-glance-material:ID][Title]]= link materializes its target on follow, and
  the graph indexes it as an edge — =C-c @= tables the headline's references,
  =C-u C-c @= its back-references.
- *Encrypted headlines* — secret regions live in =#+begin_crypt= … =#+end_crypt=
  blocks: ciphertext at rest, decrypted for editing in the materialized buffer
  and re-sealed on save (plaintext never touches disk; auto-save/backups are
  disabled).  Several blocks per headline; everything outside them stays public
  and indexable, so an encrypted headline with a plaintext link still shows up
  in =org-glance-open= and the link opens without a password.  In the buffer,
  =C-c #= is context-aware: wrap the region in a crypt block, =C-u= unwraps the
  block at point; with no region it encrypts the whole body (plaintext buffer)
  or forgets the cached password (already encrypted; TTL
  =org-glance-material-password-ttl=).  In the table, =#= toggles a whole
  headline (encrypt = one whole-body block; decrypt = open + unwrap), and
  =C-u #= re-keys.  Pre-block whole-body ciphers upgrade silently on their
  first save.
- *Repeat history* — a repeating headline (=+1d=) stays ONE headline; completing
  a repetition snapshots it under its own data dir
  (=org-glance-repeat-history-depth= newest kept, 0 = off, =t= = all;
  per-headline override via =:ORG_GLANCE_REPEAT_HISTORY_DEPTH:=).  The table marks
  repeaters in the =Rep= column; =l= on a row (or =C-c l= in the material buffer) opens a
  chosen occurrence read-only.
- *Crash-safe store* — a segmented, append-only JSONL log with atomic commits and
  an in-memory read cache.

* Install from source (development)

For hacking on org-glance, clone the repo and load it directly.  It is a
multi-directory package: =org-glance.el= is at the root, modules under =src/data=
and =src/view=; it does NOT add those to =load-path= itself, so all three must be
on it.  [[https://github.com/rails-to-cosmos/table-view][table-view]] is an external dependency from the =rails-to-cosmos= ELPA
archive (not on MELPA); the MELPA deps above are still
needed.  (=Eask= is the dev/test harness, not required to use the package.)

#+begin_src emacs-lisp
(dolist (dir '("" "src/data" "src/view"))
  (add-to-list 'load-path (expand-file-name dir "~/path/to/org-glance")))
(require 'org-glance)
#+end_src

With =straight.el= (also needs a recipe for
[[https://github.com/rails-to-cosmos/table-view][table-view]]):

#+begin_src emacs-lisp
(use-package org-glance
  :straight (org-glance :type git :host github :repo "rails-to-cosmos/org-glance"
                        :files ("*.el" "src/data/*.el" "src/view/*.el")))
(use-package table-view
  :straight (table-view :type git :host github :repo "rails-to-cosmos/table-view"))
#+end_src
