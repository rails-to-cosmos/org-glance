#+TITLE: Special org-glance properties and markers

The drawer properties, content markers and file keywords org-glance gives
meaning to.  Everything else in a headline is ordinary org.  Keep this current
when a feature adds or changes one (see [[file:../CLAUDE.md][CLAUDE.md]]).

* Headline drawer properties

Set in a headline's =:PROPERTIES:= drawer.

- =ORG_GLANCE_ID= — the headline's stable store key (content-addressed data
  dir, WAL id).  Capture generates it; a headline without it is not tracked.
  Path-safety-checked before it touches the filesystem.  Read by
  =org-glance-headline:id=.  Concealed (never deleted) in material buffers via
  =org-glance-material-hidden-properties=.
- =ORG_GLANCE_PROJECT_DIR= — a project directory for =org-glance-llm= to open its
  session in, instead of the content data dir.  Set/clear from a materialized
  buffer with =C-c d= (=org-glance-material:set-project-dir=); read by
  =org-glance-llm--dir=.  =~= is expanded; the dir is created if missing.
- =ORG_GLANCE_TITLE= (or =TITLE=) — overrides the displayed title; otherwise the
  heading text is used.  Read in =org-glance-headline--from-element=.
- =ORG_GLANCE_REPEAT_HISTORY_DEPTH= — per-headline override of
  =org-glance-repeat-history-depth=: an integer (=0= disables) or =t=/=inf=
  (unlimited).  Read at repeat completion by
  =org-glance-material--history-depth=.
- =ORG_GLANCE_HASH= — legacy content-hash property; no current code writes it.
  Stripped before hashing (=org-glance-headline:hash-ignore-properties=),
  skipped by the tag-config lint, concealed and edit-reverted in material
  buffers.  Internal — do not set by hand.

* Tag-config files

One file per tag, =<store>/config/tags/<tag>.org= (see
[[file:../src/data/org-glance-tag-config.el][org-glance-tag-config]]); the file
name is the tag.  Every pragma is OPTIONAL — a freshly stubbed file carries only
the capture template (comments plus a =* %?= heading), so the tag inherits every
global default.  Pragmas, above the first =*= heading, override those defaults:

- =#+TITLE:= — the tag's human label (default: the tag name).
- =#+TODO:= — the tag's todo cycle, e.g. =TODO READING | READ ABANDONED=; an
  ordinary org file keyword, prepended to the capture buffer and emitted in the
  overview/agenda header (absent = the global =org-todo-keywords=).
- =#+CATEGORY:= (planned) — agenda category, default the tag name.
- =#+AUTHOR:= (planned).

The body from the first =*= heading is the =org-capture= entry skeleton; capture
fills that heading's text with the instance title + =:tags:= (a bare =* %?= is
the default entry).

Legacy: a single =config/tags.org= with =:TAG:=-keyed level-1 headlines and a
=:TODO_KEYWORDS:= drawer is split into per-tag files on graph open and backed
up as =tags.org.bak=.  A residual =:TAG:=/=:TODO_KEYWORDS:= drawer in a per-tag
file is ignored (=org-glance-tag-config-lint= flags it).

* Content markers (in a headline's body)

- =#+begin_crypt= … =#+end_crypt= — a secret region: ciphertext at rest,
  plaintext only in the material buffer.  Several per headline; text outside
  stays public and indexable.  See [[file:proposals/2026-07-17-crypt-blocks.done.org][crypt-blocks]].
- =#+begin_pin= … =#+end_pin= — a pinned block: kept when a repeated headline is
  trimmed on completion (=org-glance-repeat-history-depth= non-zero), so it survives
  in the live headline while the trimmed history lives on in the occurrence
  snapshot (see [[file:proposals/2026-07-18-repeat-occurrences.done.org][repeat-occurrences]]).

* Link types (in a headline's body)

Org link types org-glance registers; all follow with the ordinary =C-c C-o=.

- =org-glance-material:ID= — a relation edge: following materializes ID.  An
  optional =?kind=KIND= suffix types the reference (a book's =author= vs
  =editor=, =roasted by=); the kind annotates the edge, and following behaves
  the same either way.  Kinds
  are stored dash-slugged and downcased in the path (=?kind=roasted-by=); the
  spaced form is display-only (prose prefix, prompts, column headers).
  Inserted by =@= in a material buffer; =C-u @= asks the kind and prefixes the
  link with it as prose (=roasted by [[…][Manhattan]]=).  The store derives the =relations=
  metadata from these links on save/reindex.  See
  [[file:proposals/2026-07-18-relations.done.org][relations]].
- =org-glance-visit:ID= — legacy materialize link; still followed, and counts
  as a kindless relation edge.
- =org-glance-open:ID= — follows the link INSIDE headline ID.
- =org-glance-overview:TAG[?KEY=VALUE&…]= — opens the overview (in
  =org-glance-overview-default-view=).  A bare TAG merges the ambient
  =org-glance-filter-spec=; a =?=-qualified path is explicit — exactly the
  filter it states.  KEYs are the filter-table keys (=done=, =state=,
  =title-contains=, =tags=, =id-any=, =refers-to=, =priority=, …); lists
  comma-separated, booleans =t=/=nil=, planning keys =present=/=absent=,
  priority a letter; TAG =all= or empty
  adds no tag constraint; =:where= is not linkable.  E.g.
  =[[org-glance-overview:book?done=nil&state=READING][Reading now]]=.  Parsed
  by =org-glance-filter:from-link-path=.

* File keywords

- =#+TODO:= — an ordinary org file keyword.  Capture and the overview/agenda
  emit it from a tag's =#+TODO:= cycle (its config file) so org renders, faces
  and cycles those states natively; the table shows a coloured =#+TODO:= line of
  the graph's live states in its header (always visible).

* Crypt blocks and materialization

A headline whose blob carries sealed =#+begin_crypt= blocks materializes
AS-IS: the ciphertext is what you see, no password is requested, and the
buffer is NOT hardened (nothing secret is in it yet).  Decrypt explicitly
-- =C-c u= (=org-glance-material:decrypt=), or open with
=org-glance-material:open ... :decrypt t= (the transient's =-d= switch)
-- at which point the buffer hardens and the seal/unseal save round-trip
wires up (invariant 14 unchanged: plaintext never reaches disk).

* Body date interval

=<from>--<to>= — an org-native active timestamp range in the body.  The
FIRST one is the headline's interval: projected into =range= metadata
(=org-glance-headline--content-facts=), rendered verbatim in the overview
heading and as the table's =Interval= column.  Set with =C-c i= in the
material buffer (=C-u C-c i= removes).  Title, planning-line, drawer and
LOGBOOK ranges never count (the scan starts after the heading's
meta-data), and =C-c i= never touches a range inside a crypt block --
sealed ones are invisible to the index (invariant 14), decrypted ones are
secret content.
